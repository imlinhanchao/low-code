# 国际化配置

设计器及其组件库内置了国际化支持，支持多语言显示与配置。

## 语言初始化 {#initialization}

设计器默认根据浏览器语言自动初始化 locale。
- 如果浏览器语言为中文，则默认使用 `zh-CN`。
- 如果浏览器语言为非中文，则默认使用 `en-US`。

## 国际化 API {#i18n-api}

通过向 `LcDesigner` 或 `LcRenderer` 组件设置 `ref`，你可以调用以下 API 进行语言管理：

### `setLocale(lang: string)` {#setLocale}
切换当前显示语言。

```javascript
// 切换到英文
designerRef.value.setLocale('en-US')

// 切换到中文
designerRef.value.setLocale('zh-CN')
```

### `addLocale(lang: string, messages: object)` {#addLocale}
添加、扩展或覆盖现有的语言包。

```javascript
// 添加日语支持
designerRef.value.addLocale('ja-JP', {
  designer: {
    componentLib: 'コンポーネントライブラリ',
    canvas: 'キャンバス',
    propsPanel: '属性',
    // ...
  }
})
```

- **参数 `lang`**: 语言标识符（如 `en-US`, `zh-CN`, `ja-JP` 等）。
- **参数 `messages`**: 语言包对象。

---

## 组件定义中的国际化 {#component-i18n}

在定义自定义组件时，推荐在 `label` 属性中使用国际化对象：

```typescript
export const MyComponent = {
  name: 'my-custom-input', // 内部标识符，建议始终用英文
  label: {
    'zh-CN': '我的输入框',
    'en-US': 'My Input'
  },
  props: [
    {
      name: 'placeholder',
      label: {
        'zh-CN': '占位符',
        'en-US': 'Placeholder'
      }
    }
  ]
}
```

## 语言解析优先级 {#i18n-priority}

当系统尝试翻译一个文本对象（Label Object）时，遵循以下优先级：
1. **当前语言 (locale)**: 如 `en-US`。
2. **默认中文 (`zh-CN`)**: 备选方案。
3. **默认英文 (`en-US`)**: 第二备选方案。
4. **对象中的第一个键值**: 兜底方案。
5. **字符串本身**: 如果 `label` 是字符串而非对象，则直接显示。
