# Exported Interfaces & Components

## Components

### LcDesigner (Designer)

The main form designer component.

#### Props

| Property | Type | Default | Description |
| :--- | :--- | :--- | :--- |
| \`v-model\` | \`FormSchema\` | \`{ widgets: [] }\` | Form Schema data generated by the design. |
| \`components\` | \`ComponentGroup[]\` | \`[]\` | List of components registered in the designer. |
| \`locale\` | \`string\` | (Browser language) | Initial language, e.g., \`zh-CN\`, \`en-US\`. |

#### Common Props

All components in the designer have the following built-in common properties (located at the top of the properties panel):

| Prop Name | Type | Description |
| :--- | :--- | :--- |
| \`ID\` | \`string\` | Unique identifier for the component instance (i.e., field name). |
| \`CSS Classes\` | \`string\` | \`class\` bound to the component's root element. |
| \`Hidden\` | \`boolean\` | Whether to hide during rendering (no space occupied). |
| \`Read Only (Text)\` | \`boolean\` | When true, non-layout components with models will render as \`<span>\` plain text, taking the value of modelValue or the first model (Key is \`modelonly\`). |

#### Methods (Exposed)

| Method Name | Arguments | Description |
| :--- | :--- | :--- |
| \`setLocale\` | \`(lang: string) => void\` | Switches the current designer language. |
| \`addLocale\` | \`(lang: string, messages: any) => void\` | Adds a custom language pack. |
| \`getLocale\` | \`() => string\` | Gets the currently active language. |

### LcRenderer (Renderer)

Used to render the designed Schema into an actual form.

#### Props

| Property | Type | Default | Description |
| :--- | :--- | :--- | :--- |
| \`schema\` | \`FormSchema\` | (Required) | Schema data generated by the designer. |
| \`components\` | \`ComponentGroup[]\` | \`[]\` | List of components required for rendering (must match the designer). |
| \`v-model\` | \`Record<string, any>\` | \`{}\` | The actual data object bound to the form. |
| \`update:schema\` | \`(schema: FormSchema) => void\` | - | Triggered when component properties are updated via \`\$setProps\`, used to sync the latest Schema. |
| \`global\` | \`Record<string, any>\` | \`{}\` | Global shared data, accessible via \`\$global\` in scripts. |
| \`modelonly\` | \`boolean\` | \`false\` | Global read-only toggle. When true, forces all input components into plain text rendering mode. |
| \`disabled\` | \`boolean\` | \`false\` | Global disabled toggle. When true, forces \`disabled: true\` onto all component properties. |

## Methods

### addIconifyAPIProvider

Registers an Iconify icon provider with the designer, supporting the use of Iconify icons in component definitions.

| Argument | Type | Description |
| :--- | :--- | :--- |
| \`provider\` | \`(name: string) => Promise<string>\` | Icon provider function, receives an icon name and returns a Promise resolving to an SVG string. |
| \`customConfig\` | \`Partial<IconifyAPIConfig>\` | (Optional) Custom configuration items, supporting \`prefix\` (icon name prefix), etc. |

Refer to the [Iconify API Provider Documentation](https://iconify.design/docs/icon-components/svg-framework/add-api-provider.html) for details.
