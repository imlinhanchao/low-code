# Internationalization

The designer and its component library have built-in internationalization support, allowing for multi-language display and configuration.

## Language Initialization {#initialization}

The designer automatically initializes the locale based on the browser's language by default.
- If the browser language is Chinese, it defaults to `zh-CN`.
- If the browser language is not Chinese, it defaults to `en-US`.

## Internationalization API {#i18n-api}

By setting a `ref` on the `LcDesigner` or `LcRenderer` component, you can call the following APIs for language management:

### `setLocale(lang: string)` {#setLocale}
Switches the current display language.

```javascript
// Switch to English
designerRef.value.setLocale('en-US')

// Switch to Chinese
designerRef.value.setLocale('zh-CN')
```

### `addLocale(lang: string, messages: object)` {#addLocale}
Adds, extends, or overrides existing language packs.

```javascript
// Add Japanese support
designerRef.value.addLocale('ja-JP', {
  designer: {
    componentLib: 'コンポーネントライブラリ',
    canvas: 'キャンバス',
    propsPanel: '属性',
    // ...
  }
})
```

- **Argument `lang`**: Language identifier (e.g., `en-US`, `zh-CN`, `ja-JP`, etc.).
- **Argument `messages`**: Language pack object.

## i18n in Component Definitions {#component-i18n}

When defining custom components, it is recommended to use internationalization objects in the `label` property:

```typescript
export const MyComponent = {
  name: 'my-custom-input', // Internal identifier, recommended to always use English
  label: {
    'zh-CN': '我的输入框',
    'en-US': 'My Input'
  },
  props: [
    {
      name: 'placeholder',
      label: {
        'zh-CN': '占位符',
        'en-US': 'Placeholder'
      }
    }
  ]
}
```

## Language Resolution Priority {#i18n-priority}

When the system attempts to translate a text object (Label Object), it follows this priority:
1. **Current Language (locale)**: e.g., `en-US`.
2. **Default Chinese (`zh-CN`)**: Fallback.
3. **Default English (`en-US`)**: Secondary fallback.
4. **The first key-value pair in the object**: Final fallback.
5. **The string itself**: If `label` is a string rather than an object, it is displayed directly.
